# Finance Pal — Project Notes

This is a private home application for Mia and Sebastian.

## Pull Request Workflow

- **Never create draft PRs.** Always create PRs as ready-to-merge.
- **Merge immediately** after creating the PR — do not wait for approval.
- Use squash merge to keep the history clean.
- The default branch is `master`.

## Coding Principles

- No features beyond what was asked.
- No abstractions for single-use code.
- Touch only what you must — don't refactor adjacent code.
- Match existing style.
- State assumptions explicitly before implementing; ask if uncertain.
